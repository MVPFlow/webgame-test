# AI Rules (One-Tap Dodge)

- Mantener **TypeScript estricto** y evitar `any`.
- Usar **Phaser 3** como engine principal salvo una razón técnica fuerte.
- No generar arte nuevo: usar **packs de Kenney** y documentar la procedencia.
- Código limpio y directo: comentarios solo si aportan contexto no obvio.
- Priorizar cambios pequeños y verificables; actualizar documentación cuando cambie el comportamiento.
- Mantener scripts de `npm` simples (`dev`, `build`, `preview`).
- Toda nueva feature debe incluir criterios de aceptación en `docs/TODO_MVP.md` si aplica.
